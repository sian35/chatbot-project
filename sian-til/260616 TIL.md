### 새로 배운 내용 : 고급 검색 기법
#### Hybrid Search
- 임베딩 기반 의미 검색과 키워드 기반 검색을 동시에 수행하고 두 점수를 합해 최종 순위를 결정하는 검색 기법
- Dense vector 검색과 Sparse vector 검색을 모두 이용한다.
- Dense vector 검색
	- 유사도를 계산하여 찾는다.
- Sparse vector 검색
	- 특정 키워드가 정확히 일치하는 문서를 찾는다

#### Query Expansion
- 원본 질문을 재구성하거나 용어를 추가하여 검색 범위를 넓히는 기법
- Pre-retrieval 단계에 적용된다.
- 주요 확장 방식
	- 유의어 추가
	- Multi-Query
		- 원본 질문을 여러 관점의 질문으로 다시 작성
	- Step-back Prompting
		- 구체적인 질문을 더 일반적인 상위 질문으로 다시 작성

#### Reranking
- 1차 검색에서 얻은 후보 문서들을 정밀 모델로 다시 평가하여 순위를 재조정하는 기법
- Bi-Encoder
	- 1차 검색
	- 문서 임베딩을 미리 계산해 둘 수 있어서 빠른 검색이 가능하다.
	- 질문과 문서를 따로 처리하기 때문에 어떤 관계를 가지는지 세밀히 반영하기 어렵다.
- Cross-Encoder
	- Reranking
	- 질문과 문서를 하나의 입력으로 넣는 기법
	- 질문-문서 쌍의 실제 관련성 점수
	- 정밀한 판단이 가능하지만 느리다.
	- 추려진 후보들로만 이용

#### HyDE (Hypothetical Document Embeddings)
- LLM으로 가상 답변을 생성하고, 그 답변을 임베딩하여 문서 검색하는 기법
- 정답을 미리 생성하는 것이 아니라, 더 잘 검색되도록 검색용 표현을 바꾸는 것.
- Pre-retrieval 에 해당됨.
- 핵심
	- 짧은 질문보다는 긴 답변 형태가 실제 문서와 임베딩 공간에서 가깝다.
	- 갖고 있는 문서는 대부분 답변 형태이다.
- 단점
	- LLM 추가 호출
	- 가상 답변 방향 오류
	- 구체적 질문에서는 효과적이지 못함
	- 일괄 적용하기에는 비효율적이다.


#### Conversational RAG
- 이전 대화 기록을 고려해 질문을 재구성하고, 재구성된 질문으로 검색과 답변을 수행하는 **"멀티턴 RAG 시스템"**
- 핵심
	-  질문의 맥락
	- 대화 기록은 검색 입력이 아니라 **질문 재구성의 참고자료**로만 사용한다.
- 동작 흐름
	- 대화 기록(질문-답변 쌍)을 수집
	- **질문 재구성**
	- Retrieval
	- Generation
- 한계점
	- 대화가 길어지면 Context Window를 많이 차지한다.
	- LLM 추가 호출
- 주의할 점
	- 지시 따르기가 핵심인 기능에서는 모델 크기가 성능을 직접적으로 좌우한다. 
	- 따라서 지시 따르기 성능이 안정적인 모델을 질문 재구성에 사용한다.
	- Gemini, GPT-4o mini, Claude Haiku
