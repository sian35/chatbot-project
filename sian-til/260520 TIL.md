### 새로 배운 내용
#### Database: "구조화" 해놓은 데이터 "모음"
- 정형 데이터 (Structured Data)
- 자료를 어떻게 구조화 할 것인가?
#### RDB (Relational Database)
- 상호 연관된 데이터를 테이블 형태로 저장하고 제공하는 데이터베이스

#### 정규화: RDB 설계에서 중복을 최소화하게 구조화하는 프로세스
- 이상현상을 없애기 위해서 테이블을 나누는 작업
- 사용 이유: 무결성을 유지하고 저장 용량을 줄이기 위해
- Anomaly (이상 현상)
- 제1정규형~ 제5정규형 (교재에서는 3정규형까지만)

#### SQL : 구조적 데이터 질의 언어
- 사용 이유: 데이터베이스의 데이터를 조회, 추가, 수정, 삭제하기 위해
#### ERD(Entity Relationship Diagram)
- 설계도
- 사용 이유: 데이터베이스의 구조를 시각화하여 관계와 구조를 명확히 이해하기 위해
#### Index : 데이터베이스의 조회 속도를 높여주는 자료구조
- 인덱스가 없으면 Full-scan 해야해서 매우 느리다.
- B-tree
	- 대량의 데이터를 효율적으로 검색하기 위해 설계된 트리 자료 구조

#### Full-Text Index
- 텍스트 검색에서 문서 또는 전문 데이터베이스를 검색하는 기법
- 검색 엔진 (Elastic Search)

#### Transaction
- 데이터베이스 관리 시스템에서 상호작용의 단위
- 여러개의 데이터베이스 연산을 하나의 작업 단위로 묶어 처리하는 것
- e.g.) 송금

#### NoSQL (Not only SQL)
- SQL만을 사용하지 않는 여러 유형의 데이터베이스
- 대용량의 데이터를 효과적으로 관리하기 위해 설계됨
