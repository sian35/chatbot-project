### 새로 배운 내용
#### HTTP(HyperText Transfer Protocol)
- 서버와 클라이언트가 서로 데이터를 주고 받기 위해 사용되는 통신 규약

#### HTTP message
- 그러한 통신의 기본 단위
- 메시지의 종류 : 요청 메시지와 응답 메시지

#### HTTP 요청 메소드
- 웹 서버에게 어떤 작업을 수행하길 원하는지 알리는 명령으로 구체적인 작업을 명확히 지시하기 위해서 사용

#### HTTP 상태 코드
- 3자리의 숫자로 요청의 처리 상태를 나타내는 코드
- 100번대는 정보, 200번대는 성공, 300번대는 리디렉션, 400번대는 클라이언트 오류, 500번대는 서버 오류

#### HTTP URL
- 주소를 의미하며 리소스의 위치와 종류를 나타냄
- URL을 사용하는 이유는 웹 상의 리소스를 식별하고 접근하기 위해서

#### HTTP 경로 변수와 쿼리 스트링
- 쿼리 스트링은 일반적으로 URL 끝에 “?” 기호를 붙이고 그 뒤에 key 와 value 형태로 데이터를 전달하며 웹 페이지에 대한 추가 정보를 서버에 전달하기 위해 사용되는 문자열
- 경로 변수는 경로 부분에 포함된 특정 자원을 식별하는 변수로 특정 자원에 접근하기 위해 사용되며 리소스, 식별자 등을 URL 경로에 포함시켜 표현

#### JSON
- 구조화된 데이터를 표현하기 위한 문법을 의미하며 가볍고 구조화된 데이터 형식으로 데이터를 주고 받기 위해 사용됨
- 대체로 Dictionary 형태나 배열로 사용됨

#### REST API
- 자원을 이름으로 식별하고, 자원의 상태를 구조화된 형식으로 표현하여, 서버와 클라이언트 간에 정보를 교환하는 방식인 REST를 사용하는 API를 의미한다.
- 이를 사용하는 이유는 일관된 통신 규칙을 통해 업무 효율을 높이고 요청의 목적을 쉽게 이해하기 위함이다.
