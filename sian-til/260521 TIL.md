### 새로 배운 내용
#### 구조 개선
- 구조를 개선하는 이유가 있어야 한다.
- 그러한 구조를 선택한 이유를 어떻게 설명할지 & 잘 된 선택인지 어떻게 입증할지
- 현시점에는 경험이 부족하기 때문에 남의 경험을 잘 들여와서 내 것으로 만들 줄 알아야한다.

#### 디자인 패턴 : 면접 단골 질문
- 디자인이 좋다는 것은 설계가 좋다는 뜻
- 패턴 : 일정한 형태를 의미. 반복적이지 않아도.
- 자주 사용하는 설계 형태를 정형화하여 유형별로 템플릿화 한 것
- 유사한 상황에서 구조적인 문제를 해결하고 재사용성을 높이기 위해 사용한다.
- 구조적인 문제가 있다 -> 복잡도가 올라갔다.
	- 복잡도를 낮추기 위해서는?
		- 분리, 쪼개기
- 리팩토링
	- 기능의 변화 없이 구조만 변화

#### Fetch
- 웹에서 데이터를 비동기적으로 요청하고 수신하는 과정
- 서버에 접속해서 정보를 보내고 획득. 즉, 통신한다는 뜻
- 사용 이유
	- 지시된 작업을 정확히 수행하기 위해 필요한 정보를 불러오기 위해
	- 프론트엔드에서 백엔드로 데이터를 요청할 때 필수적으로 사용

#### CORS
- <-> Same Origin: 프로토콜(scheme), 호스트 이름, 포트가 모두 일치할 때 동일 출처로 간주하는 것
	- 접속의 통제 주체인 **브라우저의 정책** (모든 브라우저의 표준)
- 다른 오리진(Cross Origin)의 접속을 허용하겠다고 의사표시를 하는 주체는 **서버 정책**
- [CORS 강의교재](https://www.notion.so/adapterz/CORS-196394a48061808f8e4fcbb2b3ec076a)
- Simple Request
- Preflight Request
